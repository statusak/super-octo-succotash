# ASP.NET Application: Event Manager

Пример веб‑приложения на ASP.NET для управления мероприятиями с RESTful API и документацией Swagger.

## Функциональность

В приложении реализованы:

1. **Модель `Event`** — представляет данные о мероприятии со следующими полями:
   * `Id` — уникальный идентификатор;
   * `Title` — название мероприятия;
   * `Description` — описание (может быть пустым);
   * `StartAt` — дата и время начала;
   * `EndAt` — дата и время окончания.

2. **Интерфейс `IEventService`** — определяет контракт для работы с мероприятиями, включая методы:
   * `GetAll(FilterEvent filterEvent, int page, int pageSize)` - получить все мероприятия при пагинации и фильтре;
   * `GetAll(int page, int pageSize)` — получить все мероприятия при пагинации;
   * `GetEventById(int id)` — получить мероприятие по ID;
   * `CreateEvent(Event @event)` — создать новое мероприятие;
   * `UpdateEvent(int id, Event @event)` — обновить мероприятие;
   * `DeleteEvent(int id)` — удалить мероприятие.

3. **RESTful API для управления событиями** — контроллер `EventsController` предоставляет следующие эндпоинты:
   * `GET /events` — список мероприятий с фильтрацией и пагинацией:
     * Параметры фильтрации:
       * `title` (`string`, опциональный) — поиск по названию (регистронезависимый, частичное совпадение);
       * `from` (`DateTime`, опциональный) — события, которые начинаются не раньше указанной даты;
       * `to` (`DateTime`, опциональный) — события, которые заканчиваются не позже указанной даты.
     * Параметры пагинации:
       * `page` (`int`, опциональный, по умолчанию `1`) — номер страницы;
       * `pageSize` (`int`, опциональный, по умолчанию `10`) — количество элементов на странице.
   * `GET /events/{id}` — мероприятие по ID;
   * `POST /events` — создание нового мероприятия;
   * `PUT /events/{id}` — обновление мероприятия;
   * `DELETE /events/{id}` — удаление мероприятия.

4. **Интеграция Swagger/OpenAPI** — автоматическая генерация интерактивной документации API


5. **Описание для Swagger** — все эндпоинты имеют подробные комментарии:
    * описания методов;
    * примеры запросов и ответов;
    * коды HTTP‑статусов;
    * параметры и их типы.

6. **Глобальная обработка ошибок** — реализован middleware `GlobalExceptionHandlingMiddleware` для перехвата исключений и возврата структурированного ответа в формате **Problem Details (RFC 7807)**, содержащего:
   * код статуса HTTP;
   * сообщение об ошибке;
   * детали ошибки (при наличии).

7. **Валидация конфигурации**:
    * `ValidateScopes` — проверка корректности внедрения зависимостей (DI) при запуске приложения;
    * `ValidateOnBuild` — проверка конфигурации сервисов при сборке приложения.

8. **Юнит‑тесты** — создан отдельный тестовый проект `EventServiceTest` с использованием xUnit, покрывающий:
   * успешные сценарии (создание, получение, обновление, удаление, фильтрация, пагинация);
   * неуспешные сценарии (некорректные ID, некорректные данные, неверные даты).

## Требования

* .NET 10.0 или выше;
* текстовый редактор или IDE (например, Visual Studio, VS Code);
* браузер для доступа к Swagger UI.

## Запуск приложения

1. Клонируйте репозиторий:
```bash
git clone https://github.com/statusak/super-octo-succotash.git
cd super-octo-succotash
```

2. Восстановите зависимости:
```bash
dotnet restore
```

3. Запустите приложение:
```bash
dotnet run
```

## Запуск Unit тестов
```bash
dotnet test
```
